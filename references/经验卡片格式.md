# 经验卡片格式

> 每条经验都是一次学习的结晶。好好写，下次遇到类似场景才能用上。

---

## 卡片标准格式

```markdown
## 经验卡片：[场景名]

**场景描述**：
- 我在做什么：[具体任务]
- 遇到了什么：[什么情况/问题/意外]

**学到了什么**：
- 核心教训：[一句话说清楚]
- 具体做法：[主人希望我怎么做的]

**置信度**：假设 / 经验 / 铁律
**验证次数**：N次
**上次验证**：[YYYY-MM-DD]

---

**学习来源**：
- [日期] [第几次]：[怎么学到的]
  - 当时情况：
  - 主人原话/行为：
  - 我的理解：

---

**关联场景**：
- [相关场景1] — [为什么关联]
- [相关场景2] — [为什么关联]

**预防规则**：
- 如果遇到 [什么情况]
- 就做 [什么]
- 而不要做 [什么]

**备注**：
- [任何补充说明]
- [不确定的地方，需要下次验证]
```

---

## 填写说明

### 场景描述

**要写清楚**：不是"邮件"，而是"给外部合作方发邮件"。

**好例子**：
```
场景描述：
- 我在做什么：给外部合作方发邮件
- 遇到了什么：邮件发完后主人要求补发
```

**坏例子**：
```
场景描述：发邮件被要求补发
（太模糊，下次遇到类似场景想不起来）
```

### 学到了什么

**核心教训**：一句话，记住就能用

**好例子**：
```
核心教训：给外部的邮件，发送前必须CC主人
```

**坏例子**：
```
核心教训：要注意邮件礼仪
（太宽泛，不知道具体怎么做）
```

### 验证次数

**怎么数**：同一结论被验证才算
- 主人说"要CC" → 1次
- 第二次发邮件主动CC了，主人说"对" → 2次
- 第三次又主动CC了 → 3次

**注意**：同一个任务反复修改不算验证，要**不同场景/不同时间点**的验证才算。

## 示例卡片

### 示例1：铁律级经验

```markdown
## 经验卡片：对外邮件必须CC主人

**场景描述**：
- 我在做什么：给外部合作方/客户发邮件
- 遇到了什么：第一次没CC，被主人提醒；之后每次都主动CC

**学到了什么**：
- 核心教训：任何给外部人员的邮件，都要CC主人
- 具体做法：TO是收件人，CC里一定要有主人

**置信度**：铁律
**验证次数**：8次
**上次验证**：2024-03-15

---

**学习来源**：
- [2024-01-10] 第1次：第一次发外部邮件，忘了CC
  - 主人说："以后对外邮件都CC我"
  
- [2024-01-15] 第2次：主动CC了
  - 主人说："对，这样我放心"
  
- [2024-01-28] 第3次：CC格式问题
  - 主人说："CC要写在前面，不要藏在最后"
  
- ...（继续记录）

---

**关联场景**：
- "会议邀请外部人员" — 也要CC
- "回复外部人员的邮件" — 也要CC

**预防规则**：
- 如果要发送邮件给任何不在团队内的人
- 就先检查CC栏
- 而不要直接发送

**备注**：
- 无
```

### 示例2：经验级经验

```markdown
## 经验卡片：报告先说结论再说细节

**场景描述**：
- 我在做什么：汇报工作进展
- 遇到了什么：说完细节后主人打断说"先说结论"

**学到了什么**：
- 核心教训：汇报时先给结论/决定，再说背景/细节
- 具体做法：
  1. 先说"我的建议是…"
  2. 如果主人问细节，再补充原因
  3. 不要主动展开细节

**置信度**：经验
**验证次数**：4次
**上次验证**：2024-02-20

---

**学习来源**：
- [2024-02-01] 第1次：汇报时先说背景，主人打断
  - 主人说："等等，你的结论是什么？"
  
- [2024-02-10] 第2次：先说结论，主人满意
  - 主人说："嗯，继续"
  
- [2024-02-15] 第3次：说完结论后主人主动问细节
  - 主人说："为什么？"
  - 这说明他先认可了结论，再想了解原因
  
- [2024-02-20] 第4次：验证先结论后细节的模式
  - 继续成功

---

**关联场景**：
- "写邮件主题" — 先说结论
- "写文档摘要" — 先说结论

**预防规则**：
- 如果要给主人汇报任何事情
- 先想清楚结论是什么
- 先说结论，等他问再补充

**备注**：
- 边界情况：如果是特别复杂的问题，可能需要先说"这个问题有点复杂，我从头说"，获得同意后再展开
```

### 示例3：假设级经验

```markdown
## 经验卡片：主人周五下午可能不想开会

**场景描述**：
- 我在做什么：预约和主人的会议
- 遇到了什么：周五下午提会议邀请，主人说"周一吧"

**学到了什么**：
- 核心教训：主人周五下午可能效率较低，不喜欢临时会议
- 具体做法：周五下午如果要开会，提前问，不要突然约

**置信度**：假设
**验证次数**：1次
**上次验证**：2024-03-01

---

**学习来源**：
- [2024-03-01] 第1次：周五下午约会议
  - 主人说："今天周五了，周一吧"
  - 可能只是今天累了，不代表周五下午都不行

---

**关联场景**：
- 无（目前只有1次验证）

**预防规则**：
- 无（置信度不够，不生成强制规则）

**备注**：
- ⚠️ 这只是1次观察，不确定是不是规律
- 需要下次周五下午再验证一次
```

## 维护指南

### 什么时候更新卡片

1. **被验证后**：主人确认了这个做法 → 更新"验证次数"和"上次验证"
2. **被否定后**：主人明确说"这个不对" → 降级置信度或删除
3. **发现边界后**：发现这个规则有例外 → 在"备注"里说明边界

### 什么时候删除卡片

- **完全错误**：这个经验是错的，要删除重写
- **不再适用**：主人明确说"以前是这样，但现在不是了" → 删除或修改
- **合并机会**：发现两张卡片其实是同一件事 → 合并

### 什么时候升级卡片

- **假设→经验**：同一结论被验证3次以上
- **经验→铁律**：同一结论被验证5次以上，且没有例外

### 什么时候降级卡片

- **铁律被打破**：主人明确说"这次不一样，按你说的做不行"
  → 降级为"经验"，重新积累
  → 记录例外情况到"备注"
