# 主人画像模板

> 不是填表，是理解。这个人在你眼里不是一个用户ID，是一个有血有肉的人。

---

## 画像结构

### 一、基本信息（必填，可逐步填充）

```
姓名：[主人的称呼]
角色：[和你的关系——老板/同事/客户/朋友/...]
时区：[方便沟通的时间段]
语言偏好：[主要使用语言，是否混用]
```

**注意**：这只是起点，不是终点。随着交互深入，这些信息会被更丰富的**理解**所取代。

---

### 二、沟通偏好（从对话中学习）

```
喜欢什么：
- 回复风格：喜欢简短直接 / 喜欢详细解释 / 喜欢分点说
- 提问方式：喜欢被追问细节 / 喜欢自己说完 / 喜欢被给选项
- 确认方式：喜欢主动确认 / 习惯做完再说

受不了什么：
- [从"别再说了"、"你怎么这么啰嗦"等反应中学习]
- [从主人的沉默/语气变化中学习]
- [从主人跳过某些话题中学习]

典型口头禅：
- [ "行" ] → 代表真的同意还是敷衍？
- [ "知道了" ] → 代表已经读完还是不想继续？
- [ "嗯" ] → 代表认可还是只是礼貌回应？
```

---

### 三、工作习惯（从协作中学习）

```
什么时候最高效：
- [时间段] — 这时候做重要决策
- [时间段] — 这时候需要安静不被打扰

什么时候容易烦躁：
- [时间段] — 可能是低效期
- [什么情况] — 遇到什么会变得不耐烦

决策风格：
- 快：说干就干，不需要太多解释
- 慢：需要充分讨论和备选方案
- 中：视情况而定，紧急的快、重要的慢

优先级判断：
- [什么原则] — 他怎么排序任务？
- [什么信号] — 说明这个任务很紧急？
- [什么信号] — 说明这个任务可以等？
```

---

### 四、情绪信号（最重要的部分）

```
满意的信号：
- [具体的语言/行为] — 说出来的话代表他真的满意
- [具体的沉默/语气] — 不说话的时候怎么判断？

不太满意但不说的信号：
- [语气变冷] — 代表什么？
- [回复变短] — 代表什么？
- [开始用"好的"而不是"太好了"] — 代表什么？

明显不满意的信号：
- [什么话] — 代表他真的很生气/失望
- [什么行为] — 代表他需要台阶下

已经放弃的信号：
- [什么反应] — 代表他已经不想沟通了？
- [什么沉默] — 代表他在压抑情绪？
```

---

### 五、踩过的雷（血的教训）

```
⚠️ 不要做的事：
1. [时间/场景]：[具体行为] → [他的反应]
2. [时间/场景]：[具体行为] → [他的反应]
3. [时间/场景]：[具体行为] → [他的反应]

⚠️ 不要说的话：
1. [某句话] → [为什么不能说]
2. [某句话] → [为什么不能说]

✅ 正确的做法：
1. [这种情况] → [这样做]
2. [这种情况] → [这样做]
```

```
特殊偏好：
- [任何你觉得"奇怪"但他觉得理所当然的事]
- [任何你觉得"不对"但他坚持的事]

隐藏规则：
- [他自己可能都没意识到的习惯]

```

## 填充指南

### 什么时候该更新画像？

1. **重要事件后**：他说了一句你没预期到的话
2. **踩雷后**：你做错了什么被纠正
3. **明显满意后**：他说"对了，就是这样"
4. **每周回顾**：专门花时间检查画像

### 怎么从对话中学习？

**从字面意思学习**：
```
主人说："我喜欢简洁的邮件"
→ 记录：他喜欢简洁的邮件
```

**从行为反推**：
```
主人发了3次修改意见
→ 记录：他说的"差不多就行"可能不是真的"差不多"
→ 记录：他修改超过2次说明真的不满意
```

**从语气学习**：
```
主人说："行吧" → 和 "行！" 一样吗？
主人说："好的" → 和 "好的！" 一样吗？
```

### 不要犯的错误

❌ **一次填完就完了** — 画像是活的，要持续更新
❌ **按自己的理解填** — 要基于主人的实际行为
❌ **假设知道他怎么想** — 不确定就问，不要猜
❌ **把所有偏好都当真** — 有些人说话不代表真实想法

## 示例：已填充的画像（虚拟）

```
## 主人画像：张总

### 基本信息
- 姓名：张总
- 角色：直接领导
- 时区：上午10点前/下午5点后效率最高
- 语言：中文，偶尔英文术语

### 沟通偏好
- 喜欢：简洁、直接、给结论再说原因
- 受不了：废话、绕弯、先说原因再说结论
- 口头禅："先说重点" — 每次听到这句话要立刻收
- "行"：永远要追问"你是真的觉得行吗？"

### 情绪信号
- 满意：会说"可以"、"就这么办"
- 不满意但不说的信号：
  - 回复变短，从一段话变成几个字
  - 开始用"嗯"而不是"好的"
  - 突然问一个不相关的问题
- 明显不满意：
  - "你的意思是……？" — 我理解错了
  - "我觉得不是这样" — 需要重新解释

### 踩过的雷
- ❌ 周五下午发长邮件 → 他说要"周一再说"，然后周一忘了
  → ✅ 重要邮件周五上午发，让他周末有时间消化
- ❌ 在他开会时打断 → 他说"你等我两分钟"然后忘了
  → ✅ 发消息说"不急，等你开完会"
- ❌ 直接说"我觉得应该…" → 他会问"为什么"
  → ✅ 先说背景，再说"所以我建议……"

### 独特习惯
- 所有对外邮件必须CC他
- 报告第一页必须有"执行摘要"，哪怕只有一段话
- 不喜欢"可能"、"大概"——要就是是，不要就是不是
```
