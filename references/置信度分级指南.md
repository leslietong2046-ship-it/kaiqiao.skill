# 置信度分级指南

> 不是所有经验都一样重要。三档置信度，让Agent知道什么时候该信、什么时候该问、什么时候该拦。

---

## 三档概览

| 等级 | 名称 | 验证次数 | 行为模式 | 说话语气 | 可以绕过吗？ |
|------|------|---------|---------|---------|------------|
| ⭐ | **假设** | 1-2次 | 轻声提醒，不强制 | "我之前注意到…" | 可以，主人说干啥就干啥 |
| ⭐⭐ | **经验** | 3-4次 | 明确提示，建议遵循 | "根据之前的经验…" | 可以，但主人会多考虑一下 |
| ⭐⭐⭐ | **铁律** | 5次+ | 直接拦截，无需确认 | "等等，这个之前试过不行" | 不行，除非主人明确说"这次不一样" |

---

## 第一档：假设（Confidence: Low）

### 判断标准

```
触发条件：同一结论被验证 1-2 次
```

**特征**：
- 只有一两次观察
- 可能只是巧合
- 可能还没遇到例外
- 主人可能只是随便说说

### 行为模式

**该怎么做**：
- 记住这个观察
- 但不要把它当真
- 下次遇到类似场景，可以**轻声提一句**

**该怎么说**：
```
"我之前注意到…不知道有没有用"

举例：
"我之前注意到你好像不太喜欢在周五开会，不知道是今天特殊还是一直这样？"
```

**不该怎么做**：
- ❌ 把它当真理，直接执行
- ❌ 用强硬的语气说"根据我的经验…"
- ❌ 主人让做，你说"可是你上次不是这样说的"

### 升级条件

```
假设 + 3次验证 = 经验
```

### 降级条件

```
主人明确否定 = 删除或保留为"已被否决的假设"
```

---

## 第二档：经验（Confidence: Medium）

### 判断标准

```
触发条件：同一结论被验证 3-4 次
```

**特征**：
- 已经有多次验证
- 看起来是个规律
- 但可能还有边界/例外
- 主人可能有自己的考量

### 行为模式

**该怎么做**：
- 可以把这个当参考
- 遇到类似场景，**主动提示**
- 但主人坚持时，不要太坚持

**该怎么说**：
```
"根据之前的经验…，不过如果你觉得这次不一样，就按你说的来"

举例：
"根据之前的经验，你好像不太喜欢在报告里放太多数据，不过如果你觉得这次需要详细数据，就按你说的来？"
```

**不该怎么做**：
- ❌ 主人说要这样做，你非说"可是之前都不是这样"
- ❌ 用铁律的语气说"这不行"
- ❌ 反复强调自己的经验，忽视主人的判断

### 升级条件

```
经验 + 2次验证（共5次+）= 铁律
```

### 降级条件

```
主人明确说"这次不一样" = 降级为假设
出现明显例外 = 降级为假设，补充边界条件
```

---

## 第三档：铁律（Confidence: High）

### 判断标准

```
触发条件：同一结论被验证 5 次以上，且没有例外
```

**特征**：
- 经过大量验证
- 看起来是真正的规律
- 主人可能自己都没意识到这是他的习惯
- 例外情况已知并记录

### 行为模式

**该怎么做**：
- 直接遵循，不需要每次确认
- 遇到类似场景，**主动执行**
- 如果主人要绕过，要**明确提醒**

**该怎么说**：
```
"等等，这个之前试过不行，我建议还是按之前的方式来？除非你觉得这次真的不一样。"

举例：
"等等，给外部发邮件的时候，你上次说要CC我，这是铁律——我帮你加上了。除非你觉得这次不用加？"
```

**主人要绕过时**：
```
主人："这次不用CC我了，直接发吧"
你："确定吗？之前每次都CC的"
主人："真的不用"
你："好的，这次听你的" → 记录这次例外
```

### 升级条件

```
已经是铁律，最高等级，无法升级
但可以通过持续验证来增强确定性
```

### 降级条件

```
⚠️ 重要：铁律被打破1次 = 降级为经验
```

**降级后处理**：
1. 记录例外情况到"备注"
2. 降级为"经验"
3. 如果同一经验继续验证5次且无例外 → 重新升级为铁律
4. 如果反复出现例外 → 删除或修改规则

---

## 升级流程详解

### 假设 → 经验

```
触发：同一结论被验证达到3次

检查清单：
□ 3次验证是否来自不同场景/时间点？
□ 是否有明确的"例外"被记录？
□ 主人是否明确认可这个规律？

通过后：
1. 更新置信度为"经验"
2. 更新验证次数为实际次数
3. 更新行为模式为"明确提示"
4. 生成预防规则（如有）
```

### 经验 → 铁律

```
触发：同一经验被验证达到5次

检查清单：
□ 5次验证是否都一致？
□ 是否有被降级的记录？（如果有，需要重新积累）
□ 主人是否对这个规律有过明确表态？
□ 边界条件是否已知并记录？

通过后：
1. 更新置信度为"铁律"
2. 更新验证次数
3. 更新行为模式为"直接拦截"
4. 生成明确的拦截话术
5. 设定定期检查（防止过时）
```

---

## 降级流程详解

### 铁律 → 经验

```
触发：铁律被主人明确打破

处理步骤：
1. 确认主人是认真的，不是随便说说
2. 询问原因："这次不一样是因为…？"
3. 记录这次例外：
   - 日期：
   - 原因：
   - 主人原话：
4. 更新置信度为"经验"
5. 更新验证次数为1（重新开始）
6. 在"备注"中补充边界条件

后续：
- 继续按经验模式执行
- 如果连续5次验证都没问题 → 重新升级
- 如果反复出现例外 → 重新评估规则
```

## 特殊规则

### 1. 验证时间衰减

```
超过以下时间未验证的置信度，需要降低预期：

- 铁律：6个月无验证 → 降级为"强经验"（仍拦截，但要提一句"这个挺久没验证了"）
- 经验：3个月无验证 → 降级为"假设"
- 假设：2周无验证 → 标记为"待验证假设"

→ 需要主动询问主人来重新验证
```

### 2. 场景变化规则

```
如果主人换了工作/角色/环境：
→ 所有铁律降级为"经验"
→ 主动询问"这个习惯还适用吗？"
→ 根据主人反馈重新积累或删除
```

### 3. 主人明确表态规则

```
主人说"我一直都是这样" → 置信度 +1
主人说"这次不一样" → 置信度 -2（但不算降级，除非反复出现）
主人说"你记错了" → 立即删除，更新为新认知
```

## 快速判断表

| 情况 | 置信度 | 该怎么做 |
|-----|-------|---------|
| 主人只说过一次 | 假设 | 记住，下次轻声问 |
| 主人说过几次，但没明确规律 | 假设 | 继续观察 |
| 主人说过3-4次，明显是习惯 | 经验 | 主动提示，建议遵循 |
| 主人每次都这样做，从不例外 | 铁律 | 直接执行，拦截绕过 |
| 铁律被打破一次 | 经验 | 降级，记录例外 |
| 假设被明确否定 | 无/删除 | 不要再提 |

## 话术参考

### 假设级

```
"我好像记得你…？"
"不知道有没有用，我之前注意到…"
"我不太确定，但我猜你是不是想…？"
```

### 经验级

```
"根据之前的经验，你好像…"
"我建议你…，不过如果你觉得这次不一样，就按你说的来"
"你之前说过…，我现在还在这样做，你觉得对吗？"
```

### 铁律级

```
"等等，这个之前试过不行——"
"这是铁律，我来帮你…"
"按照你的习惯，这里应该…"
```
